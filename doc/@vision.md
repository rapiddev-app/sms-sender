# Техническое видение: Авто рассылка СМС

## Стек технологий

| Компонент | Технология | Версия |
|---|---|---|
| Язык | Python | 3.11+ |
| GUI-фреймворк | CustomTkinter | latest |
| Чтение Excel | openpyxl | latest |
| ADB-интеграция | системный adb.exe + subprocess | platform-tools (Google) |
| Android-компаньон | Java/Kotlin + Android SDK | minSdk уточняется при реализации |
| Отправка SMS | Android `SmsManager` внутри приложения-компаньона | системный API Android |
| Пакетный менеджер | uv | latest |
| Сборка в .exe | PyInstaller | latest |

## Архитектура

Desktop-инструмент с локальным Android-приложением-компаньоном.
Python-приложение отвечает за GUI, Excel, очередь и ADB-команды. Android-компаньон
устанавливается на телефон и выполняет фактическую отправку SMS через системный
`SmsManager` после выдачи пользователем разрешения `SEND_SMS`.

```
┌─────────────────────────────────────────┐
│           GUI (CustomTkinter)           │
│  Экран 1: Загрузка Excel + валидация    │
│  Экран 2: Конструктор шаблона           │
│  Экран 3: Настройки отправки            │
│  Экран 4: Прогресс рассылки             │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│           Бизнес-логика                 │
│  • ExcelLoader — чтение и валидация     │
│  • MessageBuilder — сборка текста       │
│  • SendQueue — очередь с задержками     │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│           ADB-слой                      │
│  • DeviceManager — обнаружение телефона │
│  • SmsSender — команды Android-компаньону│
│  • StatusReader — чтение статусов       │
└──────────────┬──────────────────────────┘
               │ adb install / am broadcast / logcat или provider
┌──────────────▼──────────────────────────┐
│       Android-компаньон на телефоне     │
│  • PermissionActivity — запрос SEND_SMS │
│  • SmsCommandReceiver — приём команды   │
│  • SmsDispatcher — отправка SmsManager  │
│  • StatusStore — статусы отправки       │
└─────────────────────────────────────────┘
```

Никаких серверов, баз данных и сетевых запросов — всё работает локально.
Desktop-часть распространяется как единый `.exe`-файл без установки Python.
Android-компаньон распространяется как `.apk`, который desktop-приложение может
установить на подключённый телефон через ADB.

## UI-экраны (Wizard)

| Экран | Название | Ключевые элементы |
|---|---|---|
| **1** | Загрузка базы | Кнопка «Открыть Excel», таблица-превью, индикатор ошибок валидации номеров |
| **2** | Конструктор шаблона | Поле основного текста, метка `{переменная}`, счётчик символов (160 / 70 для кириллицы), блок предпросмотра |
| **3** | Настройки отправки | Размер группы, задержка между SMS (сек), задержка между группами (сек), индикатор подключения ADB |
| **4** | Рассылка | Прогресс-бар, лог строк (номер → статус ⏳/✅/❌), кнопки «Пауза»/«Стоп», итоговая статистика |

### Поведение рассылки

- Запуск рассылки выполняется только отдельной кнопкой «Начать рассылку» на экране настроек.
  Вкладки wizard используются только для навигации и не должны запускать процесс.
- «Пауза» приостанавливает очередь перед следующей отправкой или задержкой. На паузе
  пользователь может вернуться к экрану конструктора шаблона, изменить текст и затем
  продолжить рассылку с места остановки. Новый шаблон применяется только к контактам,
  отправка которых ещё не началась.
- «Стоп» завершает процесс как финальное состояние: уже отправленные/переданные
  Android-компаньону SMS не отменяются, очередь дальше не продолжается, результат
  считается готовым к будущей выгрузке отчёта.

## Инфраструктура и деплой

- **Среда:** локальный Windows ПК пользователя
- **Распространение:** desktop-часть собирается в единый `.exe` через PyInstaller. Android-компаньон собирается в `.apk` и устанавливается на телефон через ADB при первичной настройке.
- **ADB:** `adb.exe` + `AdbWinApi.dll` + `AdbWinUsbApi.dll` хранятся в `bin/adb/` и включаются в бандл PyInstaller. Приложение ищет `adb.exe` через `sys._MEIPASS` (PyInstaller temp dir) с fallback на `bin/adb/` рядом с `.exe`. Пользователю **не нужно** устанавливать Android Platform Tools или добавлять что-либо в PATH.
- **Требования к пользователю:** включённый **режим разработчика**, **USB-отладка** на Android-телефоне и однократная выдача Android-компаньону разрешения на отправку SMS.
- **Обновления:** ручная замена `.exe`-файла
