# Авто рассылка СМС

Десктопная программа для массовой отправки персонализированных SMS через
подключённый Android-телефон по USB.

Программа работает локально: без серверов, баз данных и облачных сервисов.

## Что умеет

- Загружает контакты из `.xlsx`.
- Проверяет номера телефонов перед отправкой.
- Собирает SMS по шаблону с персональной переменной.
- Сохраняет и загружает именованные шаблоны.
- Отправляет SMS группами через Android-телефон.
- Поддерживает паузу, продолжение и остановку рассылки.
- Выгружает результат обратно в Excel со столбцом `Статус`.

## Требования

- Windows 10/11.
- Android-телефон с SIM-картой и возможностью отправки SMS.
- USB-кабель.
- Включённый режим разработчика и USB-отладка на телефоне.

Python, ADB и Android Platform Tools конечному пользователю устанавливать не нужно:
они включены в собранный `.exe`.

## Первый запуск

1. Запустите `SMS Auto.exe`.
2. Подключите Android-телефон к компьютеру по USB.
3. На телефоне подтвердите запрос USB-отладки, если Android его покажет.
4. На экране настроек приложения нажмите `Проверить ADB`.
5. Программа установит или обновит Android-компаньон на телефоне.
6. На телефоне выдайте Android-компаньону разрешение на отправку SMS.
7. Вернитесь в desktop-приложение и снова нажмите `Проверить ADB`.

Android-компаньон нужен для фактической отправки SMS через системный `SmsManager`.
Desktop-приложение устанавливает его на телефон через ADB.

## Формат Excel

Файл должен быть в формате `.xlsx`.

Используются первые два столбца:

| Столбец | Назначение |
|---|---|
| 1 | Номер телефона |
| 2 | Персональная переменная для шаблона |

Поддерживаемые форматы номеров:

- `79990000000`
- `89990000000`
- `+79990000000`

## Как отправить рассылку

1. На экране `Загрузка базы` выберите Excel-файл.
2. Проверьте превью контактов и ошибки валидации.
3. На экране `Шаблон SMS` введите текст сообщения.
4. При необходимости вставьте `{переменная}` — она будет заменена значением из
   второго столбца Excel.
5. На экране `Настройки` задайте размер группы и задержки.
6. Нажмите `Начать рассылку`.
7. На экране `Рассылка` следите за прогрессом и статусами.

## Пауза и остановка

- `Пауза` приостанавливает очередь перед следующей отправкой или задержкой.
- На паузе можно вернуться к шаблону, изменить текст и продолжить рассылку.
- Новый текст применяется только к контактам, которые ещё не начали отправляться.
- `Стоп` завершает рассылку окончательно. Уже переданные на телефон SMS не
  отменяются.

## Экспорт результата

После остановки или завершения рассылки нажмите `Выгрузить Excel`.

Программа создаст копию исходного файла со столбцом `Статус`:

| Статус | Значение |
|---|---|
| `✅` | команда отправки принята |
| `❌` | ошибка отправки |
| `⏳` | контакт не обработан |

## Где хранятся шаблоны

Шаблоны сохраняются локально:

```text
%APPDATA%/SMS Auto/templates.json
```

Если файл шаблонов повреждён, программа запустится с пустым списком шаблонов.

## Удаление

Программа не требует установки через мастер.

Чтобы удалить desktop-приложение:

1. Удалите `SMS Auto.exe`.
2. При необходимости удалите папку `%APPDATA%/SMS Auto`.
3. На телефоне удалите Android-приложение-компаньон обычным способом через
   настройки Android.

## Частые проблемы

### Телефон не найден

- Проверьте USB-кабель.
- Включите режим разработчика и USB-отладку.
- Подтвердите запрос USB-отладки на телефоне.
- Оставьте подключённым только один Android-телефон.

### Нет разрешения на отправку SMS

Откройте Android-компаньон на телефоне и выдайте разрешение `SMS`.

### Excel не загружается

- Проверьте, что файл имеет формат `.xlsx`.
- Проверьте, что в первых двух столбцах есть номера и переменные.
- Исправьте строки, которые программа показала как ошибки валидации.

## Разработка

### Установка зависимостей

```powershell
uv sync
```

### Загрузка ADB

```powershell
uv run python scripts/fetch_adb.py
```

Скрипт скачает Android Platform Tools с официального сервера Google и положит
`adb.exe`, `AdbWinApi.dll`, `AdbWinUsbApi.dll` в `bin/adb/`.

### Сборка Android-компаньона

```powershell
cd android/sms_companion
.\gradlew.bat assembleDebug
```

Ожидаемый APK:

```text
android/sms_companion/app/build/outputs/apk/debug/app-debug.apk
```

### Запуск desktop-приложения из исходников

```powershell
uv run python src/main.py
```

### Проверки

```powershell
uv run ruff check .
uv run pytest
```

### Сборка `.exe`

```powershell
uv run python build.py
```

Готовый файл:

```text
dist/SMS Auto.exe
```
